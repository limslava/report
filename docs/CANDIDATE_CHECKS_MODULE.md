# Модуль «Проверка кандидатов»

Документ фиксирует текущее состояние бизнес-процесса проверки кандидатов,
его роли, хранение анкет и уведомления.

## Назначение

Модуль нужен для передачи анкеты кандидата из HR в службу безопасности и
фиксации результата проверки без смешивания с договорным модулем.

Основной сценарий:

1. HR создаёт проверку кандидата.
2. HR прикрепляет анкету кандидата.
3. Руководитель СБ получает уведомление на email.
4. Руководитель СБ открывает карточку проверки, скачивает анкету и выбирает
   решение.
5. HR получает email-уведомление о результате проверки.

## Роли и доступ

- `head_hr`, `hr_specialist`:
  - видят раздел `Бизнес процесс` → `Проверка кандидатов`;
  - создают новые проверки;
  - обязаны прикрепить анкету при создании;
  - видят все проверки и результаты СБ;
  - скачивают анкеты.
- `security`:
  - видит раздел `Бизнес процесс` → `Проверка кандидатов`;
  - видит все проверки;
  - скачивает анкеты;
  - принимает решение по проверке.
- `admin`:
  - видит все проверки и анкеты;
  - может создать проверку как HR;
  - может принять решение как руководитель СБ.

## Решения СБ

Руководитель СБ выбирает один из трёх результатов:

- `Согласован`;
- `Согласован с замечаниями`;
- `Не согласован`.

Для решений `Согласован с замечаниями` и `Не согласован` комментарий
обязателен. Повторно принять решение по уже завершённой проверке нельзя.

## Анкеты и файлы

Анкеты хранятся отдельно от договоров:

- таблица проверок: `candidate_checks`;
- таблица файлов: `candidate_check_attachments`;
- каталог файлов: `uploads/candidate-checks/{candidateCheckId}/...`.

Разрешены файлы:

- PDF;
- DOC;
- DOCX;
- PNG;
- JPG/JPEG.

Ограничение размера одного файла: до 20 МБ.

Анкеты не вкладываются в email-уведомления. В письме отправляется ссылка на
карточку проверки, а скачивание анкеты происходит только из системы после
проверки роли пользователя.

## API

- `GET /api/candidate-checks` — список проверок с поиском и фильтром по
  статусу.
- `POST /api/candidate-checks` — создание проверки, доступно HR.
- `POST /api/candidate-checks/:id/decision` — решение СБ.
- `GET /api/candidate-checks/attachments/:attachmentId/download` — скачивание
  анкеты.

## Frontend

Страница: `/business-processes/candidate-checks`.

Интерфейс:

- таблица проверок;
- поиск по ФИО, должности, телефону и email;
- фильтр по статусу;
- создание проверки из кнопки `Новая проверка`;
- карточка по двойному клику;
- скачивание анкет из карточки;
- deep link из письма через параметр `candidateCheckId`.

## Email

Модуль использует существующую настройку SMTP:

- при создании проверки письмо уходит активным пользователям с ролью
  `security`;
- после решения СБ письмо уходит HR, который создал проверку;
- если SMTP недоступен, ошибка логируется, но бизнес-операция не откатывается.

## Границы модуля

Модуль проверки кандидатов не использует таблицы и файлы договоров. Общими
остаются только пользователи, роли, авторизация и SMTP-сервис.
