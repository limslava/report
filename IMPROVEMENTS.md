## 🔐 Безопасность

### 1. Замена eval() на безопасный калькулятор

**Проблема:** В файле [`fieldCalculations.ts`](frontend/src/utils/fieldCalculations.ts:30) использовался `eval()` для вычисления формул, что создавало серьезную уязвимость.

**Решение:** Создан модуль [`safeCalculator.ts`](frontend/src/utils/safeCalculator.ts) с:
- Лексическим анализатором (tokenizer)
- Парсером с алгоритмом сортировочной станции (Shunting Yard)
- Вычислителем обратной польской нотации (RPN)

**Преимущества:**
- ✅ Полная защита от инъекций кода
- ✅ Поддержка переменных и арифметических операций
- ✅ Валидация формул перед вычислением
- ✅ Понятные сообщения об ошибках

```typescript
// Старый подход (небезопасный)
const result = eval(expression);

// Новый подход (безопасный)
const result = safeCalculate(formula, variables);
```

## 📅 Доработки: График работы (превью) — Апрель 2026

### Что изменено в интерфейсе
- Левое меню для КТК Владивосток перестроено в формат:
  - `Диспетчерский отдел`
  - внутри: `Ежедневный отчет`, `Операционный отчет`, `График работы`
  - внутри `График работы`: `Контейнеровозы`, `Автовозы`, `Диспетчера`, `Курьеры (Оперативники)`
- Реализовано автоскрытие/раскрытие подпунктов графика.
- Обновлены заголовки страниц в синей плашке под разделы графика.

### План/Факт (Контейнеровозы)
- Добавлен режим `План / Факт` (селектор).
- Реализовано копирование `План -> Факт`:
  - если в факте нет ручных правок — копирование без подтверждения;
  - если есть ручные правки — модальное подтверждение (`Заменить` / `Отменить`);
  - доступно для `admin` и `head_ktk_vvo`.
- Логика поведения после копирования выровнена под UX-сценарий (без лишнего переключения экрана).

### Логика данных по месяцам
- Состав (водители/ТС/диспетчера/курьеры) переносится из предыдущего месяца.
- График на новый месяц создается пустым (без автозаполнения смен).
- Удаления и изменения в одном месяце не должны перетирать историю других месяцев.

### Правила расчета итогов
- `Контейнеровозы`:
  - `1` и `П` считаются как присутствие;
  - если у машины 2 водителя, в `Итого по дням` машина считается как `1`, если хотя бы у одного есть `1` или `П`.
- То же правило применено для `Автовозы` в нижней строке итогов по дням.

### Редактирование и UX
- Убрана модалка `Заполнение ячейки` при двойном клике.
- Упрощено окно `Добавить` (убраны лишние поля/подсказки по сценарию раздела).
- Поведение `Сохранить` и предупреждений о несохраненных данных выровнено:
  - кнопка активируется только при реальном изменении;
  - клик без изменения не создает `dirty-state`.
- Для горячих клавиш статусов оставлены русские соответствия (без всплывающих предупреждений).

### Сортировка
- `Контейнеровозы` и `Автовозы`: сортировка по `ФИО` и `Г/Н ТС`.
- `Диспетчера` и `Курьеры`: сортировка по `ФИО`.
- Выбранная сортировка сохраняется в `localStorage` (на устройстве/в браузере пользователя).

### Экспорт
- Добавлена выгрузка `Excel` для графиков работы по сегментам.
- Визуал выгрузки приведен к формату, близкому к рабочей проформе:
  - шапка периода, двухуровневый календарный заголовок,
  - стили статусов, итоги, легенда, заморозка панелей.
