## 🔐 Безопасность

### 1. Замена eval() на безопасный калькулятор

**Проблема:** В файле [`fieldCalculations.ts`](frontend/src/utils/fieldCalculations.ts:30) использовался `eval()` для вычисления формул, что создавало серьезную уязвимость.

**Решение:** Создан модуль [`safeCalculator.ts`](frontend/src/utils/safeCalculator.ts) с:
- Лексическим анализатором (tokenizer)
- Парсером с алгоритмом сортировочной станции (Shunting Yard)
- Вычислителем обратной польской нотации (RPN)

**Преимущества:**
- ✅ Полная защита от инъекций кода
- ✅ Поддержка переменных и арифметических операций
- ✅ Валидация формул перед вычислением
- ✅ Понятные сообщения об ошибках

```typescript
// Старый подход (небезопасный)
const result = eval(expression);

// Новый подход (безопасный)
const result = safeCalculate(formula, variables);
```

## ✅ Валидация и обработка ошибок

### 5. Централизованная валидация

**Файл:** [`validation-schemas.ts`](backend/src/middleware/validation-schemas.ts)

Создан набор схем валидации с использованием **Joi**:
- Оперативные данные
- Месячные планы
- Метрики
- Начальные остатки
- Email конфигурация
- Пользовательские данные

**Пример:**
```typescript
export const monthlyDataSchema = Joi.object({
  basePlan: Joi.number().optional().min(0),
  actual: Joi.number().optional().min(0),
  carryOver: Joi.number().optional(),
}).unknown(true);
```

### 6. Улучшенный обработчик ошибок

**Файл:** [`error-handler.ts`](backend/src/middleware/error-handler.ts)

**Улучшения:**
- 🔍 Детальное логирование с контекстом
- 🎯 Специфичная обработка типов ошибок:
  - Database Query Errors
  - Validation Errors
  - JWT Errors
  - Unique Constraint Violations
  - Foreign Key Violations
- 📊 Различные уровни детализации (dev/prod)
- 🔄 Wrapper `asyncHandler` для автоматической обработки ошибок

**Использование:**
```typescript
router.get('/data', asyncHandler(getData));
```

### 7. Middleware валидации

**Файл:** [`validate.ts`](backend/src/middleware/validate.ts)

Три типа валидации:
- `validate(schema)` - для body
- `validateParams(schema)` - для URL параметров
- `validateQuery(schema)` - для query string
